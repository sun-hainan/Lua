# 模块一：代码规范与基础

## 学习目标

- 掌握 Lua 注释的两种写法（单行/多行）
- 理解 Lua 缩进规范与代码块闭合机制
- 熟悉标识符命名规则和关键字
- 理解 nil 的语义与重要性
- 了解 Lua 模块（require）的加载机制

## 1. 注释与注释规则

### 生活类比

> 写注释就像给代码加"说明书"——你自己三个月后读代码，也会忘记当初为什么这样写。

### 概念讲解

Lua 注释在**词法分析阶段就被丢弃**，不会进入运行时，不影响性能和内存。

| 类型 | 语法 | 说明 |
|------|------|------|
| 单行注释 | `--` | 两个减号，到行尾结束 |
| 多行注释 | `--[[ ... ]]` | 适合大段说明 |
| 嵌套多行 | `--[=[ ... ]=]` | 可嵌套的多行注释 |

### 代码示例（逐行注释）

```lua
--[[
模块一：代码规范与基础
讲解：注释 / 缩进 / 标识符 / 关键字
]]

-- 单行注释示例（这是一行注释）

--[[
这是多行注释
可以跨越多行
]]

--[=[
这是另一种多行注释语法
可以嵌套 --[[ 内嵌 ]]--
]=]

-- 注释在词法分析阶段被丢弃，不进入运行时
-- type(nil) 返回 "nil"，说明 nil 是真实存在的值
print("nil type:", type(nil))       -- nil type: nil
print("nil is falsy:", not nil)     -- nil is falsy: true
```

### ASCII 图示

```
源代码流:
┌──────────────────────────────────────┐
│ -- 单行注释                          │ ← 词法分析器跳过
│ --[[ 多行                           │ ← 词法分析器跳过
│    注释区域 ]]                       │
│ local x = 10  -- 这行有注释         │ ← 行内注释也跳过
└──────────────────────────────────────┘
         ↓ 词法分析（丢弃注释）
         ↓
┌──────────────────────────────────────┐
│ local x = 10                         │
└──────────────────────────────────────┘
```

### 常见错误

```lua
-- ❌ 错误：关键字不能用作标识符
-- local if = 5      -- 语法错误

-- ❌ 错误：标识符不能以数字开头
-- local 2fast = 5   -- 语法错误

-- ❌ 混淆：nil 和 false 是不同的值
print(nil == false)    -- false（类型不同）
```

## 2. 缩进与代码块结构

### 生活类比

> Lua 的 `end` 就像一本书的"章节结束标记"，告诉读者"这里故事讲完了"。缩进只是让人类读者看起来更舒服，机器根本不在乎。

### 概念讲解

Lua **不强制缩进**，空格和 Tab 都只是空白字符。但良好的缩进习惯对代码可读性至关重要。

Lua 用 `end` 关键字闭合代码块（if/for/function 等），不使用 `{}`。

### 代码示例（逐行注释）

```lua
-- Lua 不强制缩进，但强烈建议统一使用 4 空格
local function example()
    -- if 语句块以 end 结束
    if condition then
        do_something()
        if another_condition then
            do_other()
        end  -- 内层 if 结束
    end      -- 外层 if 结束
end           -- 函数结束

-- Lua 用 end 闭合，不是 }
-- 多行表达可用 \ 续行（Lua 5.2+）
local long_string = "This is a very long string that \
continues on the next line"

-- 括号可以隐式续行（逗号分隔的元素天然跨行）
local matrix = {
    {1, 2, 3},
    {4, 5, 6},
}
```

### ASCII 图示

```
代码块结构对比：

C/Java (用 {}):
if (x > 0) {
    do_something();
}

Lua (用 end):
if x > 0 then
    do_something()
end
```

### 常见错误

```lua
-- ❌ 常见错误：忘记写 end
-- if x > 0 then
--     do_something()
-- end  -- 忘记写 end 会导致语法错误

-- ❌ 混用空格和 Tab（团队协作时会造成显示混乱）
-- 建议：项目级统一约定（推荐 4 空格）
```

## 3. 标识符与关键字

### 生活类比

> 标识符就是给变量/函数起的"名字"。和人名一样，有些名字是保留的（比如"总统"是职位名，不能用作人名）。

### 概念讲解

Lua 标识符命名规则：
- 由字母、数字、下划线组成
- **不能以数字开头**
- **区分大小写**

Lua 关键字（保留字）共 22 个：`and break do else elseif end false for function if in local nil not or repeat return then true until while goto`

### 代码示例（逐行注释）

```lua
-- 合法标识符
local snake_case = 1          -- 蛇形命名
local camelCase = 2           -- 驼峰命名
local _private = 3           -- 下划线开头（约定为私有）
local with123 = 4            -- 包含数字（但不能开头）

-- 关键字不能用作标识符
-- local if = 5              -- ❌ 语法错误
-- local end = 10           -- ❌ 语法错误

-- Lua 没有 ++/+= 这样的自增运算符
local count = 0
count = count + 1            -- 必须这样写
```

### 常见错误

```lua
-- ❌ 错误：标识符以数字开头
-- local 2nd_place = "runner-up"  -- 语法错误

-- ❌ 错误：使用关键字
-- local return = 5              -- 语法错误

-- ✅ 正确：下划线开头表示私有（约定俗成）
local _internal_state = 42
```

## 4. nil：唯一的"空值"

### 生活类比

> nil 就像一个"空盒子"——代表"这里什么都没有"，甚至连盒子本身也不存在。

### 概念讲解

`nil` 是 Lua 的**唯一空值**，表示"没有值"。它是 Lua 中第八种基本类型。

重要特性：
- 全局变量未赋值默认为 nil
- table 中访问不存在的键返回 nil
- nil 是"假"值（falsy）
- nil 用来删除变量（将该位置设为 nil）

### 代码示例（逐行注释）

```lua
-- nil 表示"没有值"
local empty = nil
print("nil type:", type(nil))    -- type 返回类型名字符串

-- nil 是 falsy（假值）
print("nil is falsy:", not nil)  -- true

-- 删除变量（将其设为 nil，垃圾回收）
local temp = 100
temp = nil                        -- temp 不再存在
print("temp after nil:", temp)    -- nil

-- table 中访问不存在的键返回 nil
local dict = {name = "Alice"}
print("dict.job:", dict.job)      -- nil（键不存在）
print("dict.name:", dict.name)    -- Alice

-- nil ~= false（类型不同，语义不同）
print("nil == false:", nil == false)  -- false
```

### ASCII 图示

```
Lua 8 种基本类型：

┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
│  nil   │ boolean│ number │ string │ function│ table │ userdata│ thread │
├────────┼────────┼────────┼────────┼────────┼────────┼────────┼────────┤
│ 空值   │ true/  │ 整数   │ 不可变 │ 一等   │ 唯一   │ C 结构 │ 协程   │
│        │ false  │ 浮点   │ 字节串 │ 公民   │ 复合   │        │        │
└────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘

nil 语义：表示"没有值"，用于删除变量、未定义的键
false 语义：明确是"假"的布尔值
```

### 常见错误

```lua
-- ❌ 混淆 nil 和 false
local not_set = nil           -- 未定义
local explicitly_false = false  -- 明确为假
if not_set == false then     -- ❌ 永远不会相等
    print("impossible")
end

-- ✅ 正确判断
if not_set == nil then        -- true
    print("value was never set")
end
```

## 5. 常量约定与模块加载

### 生活类比

> Lua 没有"const"关键字，就像一个没有锁的抽屉——技术上谁都能打开，但聪明人会挂个"私人用品请勿翻动"的牌子。

### 概念讲解

Lua 没有 `const` 关键字，所有变量都是可变的。区分方式：
- **约定**：全大写表示常量（`MAX_RETRY = 3`）
- **作用域**：`local` 而非全局变量

模块加载：`require("module_name")` 搜索 `package.path`

### 代码示例（逐行注释）

```lua
-- 约定：大写视为常量（不是语言级保证）
local MAX_RETRY = 3
local DEFAULT_TIMEOUT = 30

-- require 加载模块
local os = require("os")
local io = require("io")
local table = require("table")

-- package.path 搜索路径（; 分割）
-- 模板中的 ? 会被模块名替换
-- 添加自定义搜索路径：
-- package.path = package.path .. ";./?.lua"
```

### 常见错误

```lua
-- ❌ 误以为 Lua 有 const
-- local const pi = 3.14      -- ❌ 语法错误：const 不是关键字

-- ❌ 忘记 require
-- local json = require("json")  -- 未安装 json 库会报错
```

## 6. 五语言对比

### 代码示例

```lua
-- Lua: 注释不进入运行时，nil 是唯一空值
local msg = "hello from lua"
local empty_value = nil

-- Python:  # 注释, None 是空值
-- def msg = "hello from python"
-- empty_value = None

-- Rust: // 注释, Option<T> 替代 null
-- let msg = "hello from rust";
-- let empty_value: Option<i32> = None;

-- Go: // 注释, nil 是指针/接口/切片/map/chan/函数的零值
-- msg := "hello from go"
-- var empty_value *int = nil
```

### 对比速查表

| 特性 | Lua | Python | Rust | Go | C++ |
|------|-----|--------|------|-----|-----|
| 单行注释 | `--` | `#` | `//` | `//` | `//` |
| 多行注释 | `--[[ ]]` | `"""` `'''` | `/* */` | `/* */` | `/* */` |
| 缩进 | 不强制 | 强制（4空格） | 不强制 | gofmt 强制 | 不强制 |
| 空值 | `nil` | `None` | `None`/`Option` | `nil` | `nullptr` |
| 布尔假 | `nil, false` | `None, False, 0, ""` | `false` | `false` | `false, nullptr` |
| 代码块闭合 | `end` | 缩进 | `}` | `}` | `}` |

## 本章小结

| 特性 | Lua 行为 |
|------|---------|
| 单行注释 | `--` 编译器忽略 |
| 多行注释 | `--[[ ]]` 编译器忽略 |
| 缩进 | 不强制，建议 4 空格 |
| 关键字 | 不能直接用作标识符 |
| 空值 | `nil`（唯一，表示"没有值"） |
| 代码块闭合 | `end` 关键字 |
| 模块加载 | `require("module")`，搜索 `package.path` |
| 常量 | 无 const，用大写约定 |
