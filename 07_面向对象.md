## 学习目标

- 理解 Lua 没有 class，用 table + metatable 模拟 OOP
- 掌握封装的三种方式（闭包/约定/upvalue）

## 1. 封装：私有变量的实现

### 生活类比

> Lua 没有 class 关键字，就像没有模具的陶艺师——你得自己用泥巴（table）捏东西。封装可以用"封闭盒子"（闭包）或"贴隐私标签"（约定 _ 前缀）。

### 概念讲解

Lua 没有 class，用 table + metatable 模拟 OOP。

封装方式：
- **闭包方式**（完全私有）
- **约定方式**（`_` 前缀表示私有）

### 代码示例（逐行注释）

```lua
-- 闭包方式（完全私有）
local function create_person(name, age)
    local _name = name  -- 私有变量（闭包内可见，外部无法直接访问）
    local _age = age

    return {
        get_name = function() return _name end,
        get_age = function() return _age end,
        set_age = function(a) _age = a end,
        to_string = function()
            return string.format("Person(%s, %d)", _name, _age)
        end,
    }
end

local p = create_person("Alice", 30)
print(p:get_name())     -- Alice
print(p:to_string())    -- Person(Alice, 30)
p:set_age(31)
print(p:to_string())    -- Person(Alice, 31)

-- 约定方式（Lua 风格的对象）
local function new_counter()
    return {
        _count = 0,  -- 约定 _ 前缀为私有
        increment = function(self)
            self._count = self._count + 1
        end,
        get = function(self)
            return self._count
        end,
    }
end

local c = new_counter()
c:increment()
c:increment()
print("counter:", c:get())  -- 2
```

## 2. 继承与 metatable 链

### 生活类比

> metatable 就像"族谱"——当你在自己身上找不到东西时，就去查父母的族谱。父母身上也找不到，就继续查祖父母。Lua 的 `__index` 就是这个查找机制。

### 概念讲解

Lua 继承通过 metatable 实现：
- 子类的 metatable 指向父类
- 访问子类的键找不到时，会查找 metatable（父类）

实现方式：
- `Class:new()` 创建实例
- `setmetatable(obj, {__index = Class})`
- `__index = Class` 查找机制

### 代码示例（逐行注释）

```lua
-- 父类
local Animal = {
    name = "unnamed",
    new = function(self, name)
        local obj = {name = name}
        setmetatable(obj, {__index = self})  -- obj 的 __index 指向 Animal
        return obj
    end,
    speak = function(self)
        return self.name .. " makes a sound"
    end,
}

-- 子类（继承 Animal）
local Dog = Animal:new()  -- Dog 的 __index 指向 Animal
Dog.bark = function(self)
    return self.name .. " says woof!"
end
-- Dog 没有覆盖 speak，使用父类的

local cat = Animal:new("Kitty")
print(cat:speak())  -- Kitty makes a sound

local dog = Dog:new("Rex")
print(dog:speak())  -- Rex makes a sound（继承自 Animal）
print(dog:bark())   -- Rex says woof!
```

### ASCII 图示

```
metatable 继承链：

dog = {name = "Rex"}
     ↓
metatable = {__index = Dog}
     ↓
Dog = {
    new = ...,
    bark = function(...) ... end,
    __index = Animal
}
     ↓
Animal = {
    name = "unnamed",
    speak = function(...) ... end
}

查找 dog:speak():
1. dog.speak → nil
2. dog.__index = Dog
3. Dog.speak → nil
4. Dog.__index = Animal
5. Animal.speak → "makes a sound"
```

### 常见错误

```lua
-- ❌ 直接修改类属性影响所有实例
local Animal = {name = "unnamed"}
local cat = setmetatable({}, {__index = Animal})
local dog = setmetatable({}, {__index = Animal})
cat.name = "Kitty"
print(dog.name)  -- Kitty（被影响了！）

-- ✅ 每个实例有独立属性
local Animal = {new = function(self, name)
    local obj = {name = name}  -- 每个实例独立
    setmetatable(obj, {__index = self})
    return obj
end}
```

## 3. 多态

### 生活类比

> 多态就像"同一个命令（area），不同对象有不同反应"——圆说"我的面积是 πr²"，方说"我的面积是边长²"。说的人不同，结果就不同。

### 概念讲解

多态：同一操作对不同对象有不同行为。

Lua 通过 metatable 实现：
- 方法覆盖（子类重写父类方法）
- 运行时方法查找（`__index`）

### 代码示例（逐行注释）

```lua
local Shape = {
    area = function(self) return 0 end,
}

local Circle = Shape:new()
Circle.new = function(self, radius)
    local obj = Shape.new(self, "circle")
    obj.radius = radius
    return obj
end
Circle.area = function(self)
    return math.pi * self.radius * self.radius
end

local Rectangle = Shape:new()
Rectangle.new = function(self, w, h)
    local obj = Shape.new(self, "rect")
    obj.width = w
    obj.height = h
    return obj
end
Rectangle.area = function(self)
    return self.width * self.height
end

local shapes = {
    Circle:new(5),
    Rectangle:new(4, 6),
}

-- 多态：同一接口（area）不同实现
local total_area = 0
for _, shape in ipairs(shapes) do
    print(shape.name, "area =", shape:area())
    total_area = total_area + shape:area()
end
print("total area:", total_area)
```

## 4. 运算符重载（Metamethods）

### 生活类比

> metamethod 就像"魔法契约"——你可以和桌子（table）签约，让它在被"相加"（a + b）时自动调用你指定的函数，而不是报错或做默认操作。

### 概念讲解

元方法（metamethods）定义对象的运算符行为：

| 元方法 | 运算符 |
|--------|--------|
| `__add(a, b)` | `a + b` |
| `__sub(a, b)` | `a - b` |
| `__mul(a, b)` | `a * b` |
| `__div(a, b)` | `a / b` |
| `__eq(a, b)` | `a == b` |
| `__lt(a, b)` | `a < b` |
| `__le(a, b)` | `a <= b` |
| `__concat(a, b)` | `a .. b` |
| `__tostring(a)` | `tostring(a)` |
| `__call(a, ...)` | `a(...)` |
| `__len(a)` | `#a` |

### 代码示例（逐行注释）

```lua
local Vec2 = {
    new = function(x, y)
        return setmetatable({x = x, y = y}, {
            __add = function(a, b) return Vec2:new(a.x + b.x, a.y + b.y) end,
            __sub = function(a, b) return Vec2:new(a.x - b.x, a.y - b.y) end,
            __mul = function(a, s) return Vec2:new(a.x * s, a.y * s) end,
            __eq = function(a, b) return a.x == b.x and a.y == b.y end,
            __lt = function(a, b) return a.x < b.x or (a.x == b.x and a.y < b.y) end,
            __tostring = function(a) return string.format("Vec2(%.2f, %.2f)", a.x, a.y) end,
            __len = function(a) return math.sqrt(a.x * a.x + a.y * a.y) end,
        })
    end,
}

local v1 = Vec2:new(1, 2)
local v2 = Vec2:new(3, 4)

print("v1:", v1)              -- Vec2(1.00, 2.00)
print("v2:", v2)              -- Vec2(3.00, 4.00)
print("v1 + v2:", v1 + v2)    -- Vec2(4.00, 6.00)
print("v1 - v2:", v1 - v2)    -- Vec2(-2.00, -2.00)
print("v1 * 2:", v1 * 2)       -- Vec2(2.00, 4.00)
print("v1 == v1:", v1 == v1)  -- true
print("v1 < v2:", v1 < v2)    -- true
print("#v1:", #v1)            -- 2.24（向量的长度）
```

## 5. 接口（约定）

### 生活类比

> Lua 没有 interface 关键字，就像没有"USB 接口标准文档"——但大家约定好"四针脚、扁口"，然后照着做就行。Lua 的接口就是"约定好这个表必须有 draw 方法"。

### 概念讲解

Lua 没有显式的接口，但可以用约定和契约实现：
- 表中包含特定方法
- 类型检查函数验证
- 文档约定（duck typing）

### 代码示例（逐行注释）

```lua
-- 接口约定：必须实现 draw() 方法
local function draw_shape(obj)
    if type(obj.draw) ~= "function" then
        error("object must implement draw() method")
    end
    obj:draw()
end

-- 实现接口
local Circle = {
    draw = function(self)
        print("drawing circle")
    end,
}

local Rectangle = {
    draw = function(self)
        print("drawing rectangle")
    end,
}

draw_shape(Circle)
draw_shape(Rectangle)

-- 鸭子类型（duck typing）
local duck_like = {
    draw = function()
        print("drawing something like a duck")
    end,
}
draw_shape(duck_like)  -- 只要有 draw 方法就行
```

## 6. 多重继承

### 生活类比

> 多重继承就像"混血儿"——同时有爸爸和妈妈的基因。Lua 用 `__index` 函数手动实现这个查找过程。

### 概念讲解

多重继承：子类继承多个父类。

Lua 实现方式：
- `__index` 函数手动查找多个父类
- mixin 方式（更简单）

### 代码示例（逐行注释）

```lua
-- 父类 A
local A = {
    method_a = function() return "from A" end,
}
A.__index = A

-- 父类 B
local B = {
    method_b = function() return "from B" end,
}
B.__index = B

-- 多重继承类 C
local C = {}
C.__index = function(self, key)
    if C[key] then return C[key] end
    if A[key] then return A[key] end
    if B[key] then return B[key] end
    return nil
end

local obj = setmetatable({}, C)
print("method_a:", obj:method_a())  -- from A
print("method_b:", obj:method_b())  -- from B

-- mixin 方式（更简单）
local function mixin(obj, ...)
    for _, mix in ipairs({...}) do
        for k, v in pairs(mix) do
            if not obj[k] then
                obj[k] = v
            end
        end
    end
    return obj
end

local target = {}
mixin(target, A, B)
print("mixin method_a:", target:method_a())  -- from A
```

## 本章小结

| 特性 | Lua 实现 |
|------|---------|
| 类 | table + metatable |
| 继承 | metatable 链 + `__index` |
| 多态 | 方法查找 `__index` |
| 封装 | 闭包（完全私有）或约定（_前缀） |
| 接口 | 约定（方法存在性检查）+ duck typing |
| 运算符重载 | metamethods（`__add` 等） |
| 多重继承 | `__index` 函数手动查找或 mixin |
