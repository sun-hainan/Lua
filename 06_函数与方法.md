## 1. 函数基本定义与调用

### 生活类比

> Lua 函数就像"食谱"——你把食材（参数）送进去，按照步骤（函数体）处理，得到一道菜（返回值）。食谱本身可以被复制、传递、收藏——函数是第一等公民。

### 概念讲解

Lua 函数定义：
```lua
function name(args) ... end
```

Lua 函数是**一等公民**：
- 可以赋值给变量
- 可以作为参数传递
- 可以作为返回值
- 可以匿名

### 代码示例（逐行注释）

```lua
-- 基本函数
local function add(a, b)
    return a + b
end
print("add(3, 4) =", add(3, 4))  -- 7

-- 函数赋值给变量
local multiply = function(a, b)
    return a * b
end
print("multiply(3, 4) =", multiply(3, 4))  -- 12

-- 多返回值
local function minmax(arr)
    local min, max = math.huge, -math.huge
    for _, v in ipairs(arr) do
        if v < min then min = v end
        if v > max then max = v end
    end
    return min, max
end

local m, n = minmax({3, 1, 4, 1, 5, 9, 2, 6})
print("minmax:", m, n)  -- 1  9

-- 参数默认值
local function greet(name)
    name = name or "World"
    return "Hello, " .. name .. "!"
end
print(greet())      -- Hello, World!
print(greet("Lua")) -- Hello, Lua!

-- 可变参数
local function sum(...)
    local args = {...}
    local total = 0
    for _, v in ipairs(args) do
        total = total + v
    end
    return total
end
print("sum(1,2,3,4,5) =", sum(1, 2, 3, 4, 5))  -- 15

-- select 函数访问可变参数
local function print_args(...)
    for i = 1, select("#", ...) do
        print(string.format("arg[%d] = %s", i, tostring(select(i, ...))))
    end
end
print_args("a", "b", "c")
```

### ASCII 图示

```

调用 add(3, 4)：

│ x, y =   │ →   │ function add │
│   add(3,4│     │   (a, b)     │
└──────────┘     │   return     │
      ↑          │     a + b    │
      │          └──────────────┘
      │                 ↓

│   ({1,2,3})    │

  m=1, n=3
```

### 常见错误

```lua
-- ❌ 函数名写错（Lua 不会检查）
local function add(a, b)
    return a + b
end
print(add(1, 2))   -- 3

-- ❌ 返回值数量不匹配
-- 返回多了，多余的丢弃
-- 返回少了，不足的返回 nil
```

## 2. 函数重载：Lua 不支持

### 生活类比

> Lua 没有函数重载，就像一家餐厅只有一个"炒菜"按钮——你只能告诉厨师"我要炒这个菜"，厨师自己判断食材。如果你想要"炒番茄"和"炒牛肉"两个菜谱，就得自己想办法区分。

### 概念讲解

**Lua 不支持函数重载**（同名多个函数，后者覆盖前者）。

替代方案：
- 使用可变参数 `...`
- 使用类型检查
- 使用表分发

### 代码示例（逐行注释）

```lua
-- Lua 中同名函数会被覆盖
local function test()
    print("first version")
end
test()  -- first version

local function test()
    print("second version")
end
test()  -- second version（覆盖了）

-- 类型检查实现类似重载
local function process(arg)
    if type(arg) == "number" then
        return arg * 2
    elseif type(arg) == "string" then
        return arg .. arg
    elseif type(arg) == "table" then
        return #arg
    else
        return nil
    end
end

print("process(5):", process(5))                -- 10
print("process('hi'):", process("hi"))          -- hihi
print("process({1,2,3}):", process({1, 2, 3}))  -- 3
```

### 常见错误

```lua
-- ❌ 以为 Lua 支持重载
-- local function process(x) return x end
-- local function process(x, y) return x + y end  -- 覆盖了上面的！

-- ✅ 用可变参数区分
local function process(...)
    local n = select("#", ...)
    if n == 1 then
        -- 一个参数
    elseif n == 2 then
        -- 两个参数
    end
end
```

## 3. 递归与尾调用优化

### 生活类比

> 递归就像"俄罗斯套娃"——打开一个，里面还有一个，再打开还有...直到最小的一个。普通递归每打开一层就多占一层桌子；尾递归优化就像"把娃娃直接换成说明书"——不用占桌子了。

### 概念讲解

- **递归**：函数调用自身
- **尾递归**：递归调用是函数的最后一个操作
- **尾调用消除（TCO）**：Lua VM 支持，尾递归不会栈溢出

条件：`return func(args)` 形式

### 代码示例（逐行注释）

```lua
-- 普通递归：阶乘
local function factorial(n)
    if n <= 1 then return 1 end
    return n * factorial(n - 1)  -- 不是尾调用（有乘法）
end
print("5! =", factorial(5))  -- 120

-- 尾递归版本
local function factorial_tail(n, acc)
    acc = acc or 1
    if n <= 1 then return acc end
    return factorial_tail(n - 1, n * acc)  -- 尾调用
end
print("5! (tail) =", factorial_tail(5))  -- 120

-- 斐波那契数列
local function fibonacci(n)
    if n <= 1 then return n end
    return fibonacci(n - 1) + fibonacci(n - 2)  -- 不是尾调用
end
print("fib(10) =", fibonacci(10))  -- 55

-- 迭代版本（避免栈溢出）
local function fibonacci_iter(n)
    if n <= 1 then return n end
    local a, b = 0, 1
    for i = 2, n do
        a, b = b, a + b
    end
    return b
end
print("fib(10) (iter) =", fibonacci_iter(10))  -- 55

-- 阶乘尾调用（可以处理大数）
print("factorial(1000) =", factorial_tail(1000, 1))
```

### ASCII 图示

```
普通递归（栈增长）：
factorial(4)
  = 4 * factorial(3)
  = 4 * 3 * factorial(2)
  = 4 * 3 * 2 * factorial(1)
  = 4 * 3 * 2 * 1
  = 24

栈帧: [fact(4)] → [fact(3)] → [fact(2)] → [fact(1)]
                  (占用4层)

尾递归优化（常量栈）：
factorial_tail(4, 1)
  = factorial_tail(3, 4)
  = factorial_tail(2, 12)
  = factorial_tail(1, 24)
  = 24

栈帧: [fact(4)]  (复用同一帧)
```

### 常见错误

```lua
-- ❌ 尾调用写错（多了东西）
-- return factorial(n - 1) + n  -- ❌ 不是尾调用
return factorial_tail(n - 1, n * acc)  -- ✅ 纯尾调用

-- ❌ 在尾调用后做任何操作
-- return factorial(n - 1); print(n)  -- ❌ 尾调用后不能有操作
```

## 4. 值传递 vs 引用传递

### 生活类比

> 值传递就像"复印文件"——复印件变了，原件不变。引用传递就像"给文件加链接"——改链接指向的文件，原件也变了。

### 概念讲解

Lua 的值类型 vs 引用类型：

| 类型 | 传递方式 | 说明 |
|------|---------|------|
| nil, boolean, number, string, light userdata | **值传递** | 传递副本 |
| table, function, full userdata, thread | **引用传递** | 传递引用（共享底层数据） |

注意：字符串虽然是值类型，但内部是共享的（不可变）

### 代码示例（逐行注释）

```lua
-- 数字是值传递
local function increment(n)
    n = n + 1
    return n
end

local num = 10
print("before:", num)      -- 10
increment(num)
print("after (no change):", num)  -- 10（不改变）

-- table 是引用传递
local function add_element(t, value)
    t[#t + 1] = value
end

local arr = {1, 2, 3}
print("before:", table.concat(arr, ", "))   -- 1, 2, 3
add_element(arr, 4)
print("after (changed):", table.concat(arr, ", "))  -- 1, 2, 3, 4

-- 共享引用示例
local original = {1, 2, 3}
local copy = original  -- 共享同一个表
copy[4] = 4
print("original:", table.concat(original, ", "))  -- 1, 2, 3, 4（也变了！）

-- 浅拷贝函数
local function shallow_copy(t)
    local new = {}
    for k, v in pairs(t) do
        new[k] = v
    end
    return new
end

local original2 = {1, 2, 3}
local deep = shallow_copy(original2)
deep[4] = 4
print("original2:", table.concat(original2, ", "))  -- 1, 2, 3（不变）
print("deep:", table.concat(deep, ", "))            -- 1, 2, 3, 4
```

### 常见错误

```lua
-- ❌ 以为 table 参数不会改变原数据
local function remove_last(t)
    t[#t] = nil  -- 会改变原 table！
end

-- ✅ 需要保护时先拷贝
local function remove_last_safe(t)
    local copy = shallow_copy(t)
    copy[#copy] = nil
    return copy
end
```

## 5. 闭包（Closure）

### 生活类比

> 闭包就像"带记忆的工人"——他被派出去执行任务，但走的时候还揣着创建时给他的私人笔记本（upvalue）。即使派他出去的工厂已经下班了，他的笔记本还在，里面的数据也只有他能改。

### 概念讲解

闭包：能捕获外部变量的函数。Lua 的函数可以访问创建时存在的外部变量（upvalue）。

用途：
- 工厂函数（生成函数）
- 私有变量（封装）
- 回调函数

### 代码示例（逐行注释）

```lua
-- 简单闭包：计数器
local function counter(start)
    local count = start or 0  -- upvalue
    return function()
        count = count + 1
        return count
    end
end

local c1 = counter(0)
local c2 = counter(100)

print("c1():", c1())  -- 1
print("c1():", c1())  -- 2
print("c1():", c1())  -- 3
print("c2():", c2())  -- 101
print("c1():", c1())  -- 4

-- 工厂函数
local function multiplier(factor)
    return function(n)
        return n * factor
    end
end

local double = multiplier(2)
local triple = multiplier(3)
print("double(5):", double(5))   -- 10
print("triple(5):", triple(5))   -- 15

-- 私有变量
local function create_person(name)
    local _name = name  -- 私有变量（闭包内可见，外部无法直接访问）
    return {
        get_name = function() return _name end,
        set_name = function(n) _name = n end,
    }
end

local p = create_person("Alice")
print("name:", p:get_name())  -- Alice
p:set_name("Bob")
print("name:", p:get_name())  -- Bob
-- print(_name)  -- ❌ 访问不到
```

### ASCII 图示

```
闭包结构：

function counter(start)
    local count = start  ◄── upvalue（外部局部变量）
    return function()    ◄── 返回内部函数（闭包）
        count = count + 1
        return count
    end
end

c1 = counter(0)  c2 = counter(100)

c1 的闭包：          c2 的闭包：
┌────────────┐      ┌────────────┐
│ count = 3 │      │ count = 101│  ← 各自的 upvalue
├────────────┤      ├────────────┤
│ +1 → 4    │      │ +1 → 102   │
└────────────┘      └────────────┘

两个闭包互不影响！
```

### 常见错误

```lua
-- ❌ 循环闭包陷阱（常见错误！）
local function create_counters()
    local counters = {}
    for i = 1, 3 do
        counters[i] = function() return i end  -- 全部引用同一个 i！
    end
    return counters
end

local c = create_counters()
print(c[1](), c[2](), c[3]())  -- 4, 4, 4（不是 1, 2, 3！）

-- ✅ 正确做法：捕获当前值
local function create_counters_fixed()
    local counters = {}
    for i = 1, 3 do
        local captured = i  -- 捕获当前值
        counters[i] = function() return captured end
    end
    return counters
end
```

## 6. 高阶函数

### 生活类比

> 高阶函数就像"自动化流水线"——接收一个功能（函数）作为输入，加工处理后输出。就像流水线接收刀具，能加工不同的产品。

### 概念讲解

高阶函数：接受函数作为参数或返回函数的函数。

常用高阶函数：
- **map**：对每个元素执行操作
- **filter**：过滤元素
- **reduce**：聚合元素

### 代码示例（逐行注释）

```lua
-- map
local function map(arr, func)
    local result = {}
    for i, v in ipairs(arr) do
        result[i] = func(v)
    end
    return result
end

local nums = {1, 2, 3, 4, 5}
local doubled = map(nums, function(x) return x * 2 end)
print("doubled:", table.concat(doubled, ", "))  -- 2, 4, 6, 8, 10

-- filter
local function filter(arr, predicate)
    local result = {}
    for _, v in ipairs(arr) do
        if predicate(v) then
            table.insert(result, v)
        end
    end
    return result
end

local evens = filter(nums, function(x) return x % 2 == 0 end)
print("evens:", table.concat(evens, ", "))  -- 2, 4

-- reduce
local function reduce(arr, func, init)
    local acc = init
    for _, v in ipairs(arr) do
        acc = func(acc, v)
    end
    return acc
end

local sum = reduce(nums, function(acc, x) return acc + x end, 0)
print("sum:", sum)  -- 15

local product = reduce(nums, function(acc, x) return acc * x end, 1)
print("product:", product)  -- 120
```

## 本章小结

| 特性 | Lua 行为 |
|------|---------|
| 函数定义 | `function name(args) ... end` |
| 多返回值 | `return a, b, c` |
| 默认参数 | `arg = arg or default` |
| 可变参数 | `...` / `select()` |
| 函数重载 | 不支持，用类型检查代替 |
| 闭包 | 自然的 upvalue 机制 |
| 高阶函数 | 支持，接受/返回函数 |
| 尾调用 | 支持（TCO） |
| 值传递 | nil, boolean, number, string |
| 引用传递 | table, function, thread |
