## 1. if/else 条件语句

### 生活类比

> Lua 的 if 语句就像电影院的座位选择——"如果这个座位空，就坐下；如果是别人的，就换另一个；如果都满了，就站着"。`then` 是"好，就这么做"，`end` 是"这次选择结束了"。

### 概念讲解

Lua if 语句结构：
```
if 条件 then
    ...
elseif 条件 then
    ...
else
    ...
end
```

注意：
- 用 `then` 关键字（不是 `{`）
- 用 `end` 关键字结束（不是 `}`）
- 条件表达式使用 `and/or/not`

### 代码示例（逐行注释）

```lua
local score = 85

if score >= 90 then
    print("A")
elseif score >= 80 then
    print("B")
elseif score >= 70 then
    print("C")
else
    print("D or below")
end

-- 单行 if（不推荐，但合法）
if score >= 60 then print("passed") end

-- ❌ Lua 的 if 是语句，不是表达式
-- 无法写：local grade = if score >= 90 then "A" else "B"
-- ✅ 必须用三元替代或 if 语句
local grade
if score >= 90 then
    grade = "A"
else
    grade = "B or below"
end
```

### ASCII 图示

```
if 流程图：

        ┌─────────┐
        │ score   │
        └───┬─────┘
            ▼
      ┌───────────┐
      │score>=90? │──Yes──► 打印 "A"
      └─────┬─────┘
            │ No
            ▼
      ┌───────────┐
      │score>=80? │──Yes──► 打印 "B"
      └─────┬─────┘
            │ No
            ▼
      ┌───────────┐
      │score>=70? │──Yes──► 打印 "C"
      └─────┬─────┘
            │ No
            ▼
         打印 "D or below"
```

### 常见错误

```lua
-- ❌ 忘记 then
-- if x > 0          -- ❌ 语法错误
if x > 0 then        -- ✅
    print("positive")
end

-- ❌ 忘记 end
-- if x > 0 then
--     print("positive")
-- end  -- 忘记 end

-- ❌ 混淆 then 和 do
-- if x > 0 do        -- ❌ 语法错误
if x > 0 then        -- ✅
```

## 2. Switch/Match 的替代方案

### 生活类比

> Lua 没有 switch，就像没有自动点唱机——你得自己记住每首歌在哪，想听什么歌就自己去找。表跳转就像"点歌本"，告诉你在哪页。

### 概念讲解

**Lua 没有 switch 或 match 语句**。

替代方案：
1. **if-elseif 链**：适合范围匹配
2. **表跳转**：适合离散值映射
3. **函数表**：适合需要执行逻辑的情况

### 代码示例（逐行注释）

```lua
-- 方案1：表跳转（推荐用于离散值）
local day_name = {
    [1] = "Monday",
    [2] = "Tuesday",
    [3] = "Wednesday",
    [4] = "Thursday",
    [5] = "Friday",
    [6] = "Saturday",
    [7] = "Sunday",
}

local today = 3
print("day:", day_name[today] or "invalid")  -- Wednesday

-- 方案2：函数表（适合需要执行逻辑）
local operations = {
    ["add"]    = function(a, b) return a + b end,
    ["sub"]    = function(a, b) return a - b end,
    ["mul"]    = function(a, b) return a * b end,
    ["div"]    = function(a, b) return a / b end,
}

print("add(10, 5):", operations["add"](10, 5))    -- 15
print("mul(10, 5):", operations["mul"](10, 5))    -- 50

-- 方案3：范围匹配需要 if
local score_to_grade = function(score)
    if score >= 90 then return "A"
    elseif score >= 80 then return "B"
    elseif score >= 70 then return "C"
    else return "D" end
end

print("85 ->", score_to_grade(85))  -- B
```

### 常见错误

```lua
-- ❌ 以为 Lua 有 switch
-- switch day do        -- ❌ 不存在
-- case 1: print("Mon")
-- end

-- ✅ 正确：用表
local days = {[1]="Mon", [2]="Tue"}
```

## 3. 三种循环：while / repeat / for

### 生活类比

> while 循环就像"站在门口等开门"——先看门开了没，没开就继续等。repeat 循环就像"先敲门再说"——至少敲一次，门开了就进。

### 概念讲解

Lua 三种循环：

| 类型 | 说明 | 特点 |
|------|------|------|
| `while` | 前条件循环 | 可能一次都不执行 |
| `repeat` | 后条件循环 | **至少执行一次** |
| `for` | 数值 for / 泛型 for | 精确控制 |

**注意：Lua 没有 continue！需要用 if 跳过或 goto（Lua 5.2+）**

### 代码示例（逐行注释）

```lua
-- while 循环
local i = 1
while i <= 5 do
    print("while:", i)
    i = i + 1
end

-- repeat 循环（后条件，至少执行一次）
local j = 1
repeat
    print("repeat:", j)
    j = j + 1
until j > 5

-- 数值 for（范围）
for k = 1, 5 do
    print("numeric for:", k)
end

-- 数值 for（步长）
for k = 10, 1, -2 do
    print("numeric for step:", k)  -- 10, 8, 6, 4, 2
end

-- 泛型 for（遍历数组）
local arr = {"a", "b", "c"}
for index, value in ipairs(arr) do
    print(string.format("array[%d] = %s", index, value))
end

-- pairs（遍历字典）
local dict = {x = 1, y = 2, z = 3}
for key, value in pairs(dict) do
    print(string.format("dict.%s = %s", key, value))
end
```

### ASCII 图示

```
while vs repeat 流程图：

while:                     repeat:
┌──────────┐             ┌──────────┐
│  条件?   │             │  执行语句 │
└────┬─────┘             └────┬─────┘
     │ No                         │ No
     ▼         ┌──────────┐       ▼
   退出        │  执行语句 │   ┌────────┐
               └────┬─────┘   │条件?│
                    │  Yes    └──┬─┘
                    └──────────►│
                               退出
      先判断后执行              先执行后判断
      可能 0 次                 至少 1 次
```

### 常见错误

```lua
-- ❌ repeat 的 until 条件写错
j = 1
repeat
    j = j + 1
until j > 5  -- ✅ 是 until，不是 while

-- ❌ 在 for 循环中修改循环变量
for i = 1, 10 do
    i = i + 1  -- ❌ 不要这样做！
end
```

## 4. break 与 continue（Lua 无 continue）

### 生活类比

> break 就像"提前结束电影"，continue 就像"跳过广告"——Lua 没有跳过广告的功能，只能用 if 把广告内容包起来假装跳过。

### 概念讲解

- `break`：退出当前循环
- **continue：Lua 没有！** 用以下方式替代：
  - 用 `if` 跳过
  - 用 `goto`（Lua 5.2+）模拟

### 代码示例（逐行注释）

```lua
-- break 示例
local sum = 0
for i = 1, 10 do
    if i == 5 then break end
    sum = sum + i
end
print("sum 1..4:", sum)  -- 10

-- continue 替代（用 if 包裹）
sum = 0
for i = 1, 5 do
    if i == 3 then
        -- 跳过 3
    else
        sum = sum + i
    end
end
print("sum without 3:", sum)  -- 7

-- goto 实现 continue（Lua 5.2+）
sum = 0
for i = 1, 5 do
    if i == 3 then goto continue end
    sum = sum + i
    ::continue::
end
print("sum without 3 (goto):", sum)  -- 7

-- 嵌套循环 break（只能退出一层）
local found = false
for i = 1, 3 do
    for j = 1, 3 do
        if i == 2 and j == 2 then
            found = true
            break  -- 只退出内层循环
        end
    end
    if found then break end  -- 需要检查才能退出外层
end
print("found at i=2, j=2:", found)
```

### 常见错误

```lua
-- ❌ 以为 Lua 有 continue
-- for i = 1, 10 do
--     continue        -- ❌ 语法错误
-- end

-- ✅ 用 if 跳过
for i = 1, 10 do
    if i % 2 == 0 then
        -- 偶数，跳过
    else
        -- 奇数，处理
    end
end
```

## 5. 错误处理

### 生活类比

> 错误处理就像"买保险"——正常情况顺利通行，出了问题有人收拾。`pcall` 就像把可能出事的代码包在一个保护壳里，出事了保险公司理赔，不出事正常生活。

### 概念讲解

Lua 错误处理：

| 函数 | 说明 |
|------|------|
| `error(message, level)` | 抛出错误 |
| `assert(condition, msg)` | 断言，条件为假则抛出 |
| `pcall(func, ...)` | 保护调用，捕获错误 |
| `xpcall(func, errhandler)` | 带错误处理器的保护调用 |

### 代码示例（逐行注释）

```lua
local function divide(a, b)
    if b == 0 then
        error("division by zero", 2)  -- level=2 表示调用者级别
    end
    return a / b
end

-- pcall（保护调用）
local success, result = pcall(function()
    return divide(10, 0)
end)

if not success then
    print("caught error:", result)  -- division by zero
end

-- assert 简化
local function get_element(arr, index)
    assert(index > 0, "index must be positive")
    assert(index <= #arr, "index out of bounds")
    return arr[index]
end

local arr = {1, 2, 3}
local elem = assert(get_element(arr, 2), "failed to get element")
print("element:", elem)  -- 2

-- xpcall（带错误处理器）
local function err_handler(msg)
    print("error handler caught:", msg)
    return "handled: " .. tostring(msg)
end

local ok, result = xpcall(function()
    error("something went wrong")
end, err_handler)
print("xpcall result:", ok, result)  -- true  handled: ...
```

### 常见错误

```lua
-- ❌ pcall 返回值顺序搞错
-- pcall 返回 (success, result_or_error)
local success, err = pcall(bad_function)
if not success then
    print("error:", err)  -- ✅
end

-- ❌ 在 pcall 里 return error
-- pcall 已经捕获了，不需要再 error
```

## 6. 泛型 for 迭代器

### 生活类比

> 迭代器就像"自动取票机"——你按一下（resume），它吐出下一张票（yield），票用完了就停止。不用关心里面是怎么排序的。

### 概念讲解

泛型 for 语法：
```
for var1, var2, ... in expr do ... end
```

expr 返回：迭代器函数, 状态, 初始值

常见迭代器：
- `ipairs(t)`：遍历数组
- `pairs(t)`：遍历字典
- `io.lines(filename)`：遍历文件行
- `string.gmatch(s, pattern)`：遍历匹配

### 代码示例（逐行注释）

```lua
-- ipairs 实现原理
local function my_ipairs(t)
    return function(t, i)
        local next_i = i + 1
        if t[next_i] ~= nil then
            return next_i, t[next_i]
        end
        return nil
    end, t, 0
end

local arr = {"a", "b", "c"}
for i, v in my_ipairs(arr) do
    print(string.format("my_ipairs: arr[%d] = %s", i, v))
end

-- string.gmatch（模式匹配迭代器）
local text = "hello world lua"
for word in string.gmatch(text, "%w+") do
    print("word:", word)  -- hello, world, lua
end

-- 自定义迭代器工厂
local function range(start, stop)
    return function(state, current)
        if current >= stop then return nil end
        return current + 1, current + 1
    end, nil, start - 1
end

for i in range(1, 5) do
    print(" ", i)  -- 1, 2, 3, 4, 5
end
```

### ASCII 图示

```
泛型 for 执行流程：

for var1, var2, ... in <迭代器> do
    <循环体>
end

Step 1: 计算 <迭代器> → (函数, 状态, 初值)
Step 2: 调用 函数(状态, 初值) → (var1, var2, ...)
Step 3: 如果返回 nil，结束；否则执行循环体
Step 4: 重复 Step 2

ipairs 执行：
arr = {"a", "b", "c"}
迭代器 = (ipairs 函数, arr, 0)

第一次: ipairs(arr, 0) → (1, "a") → 执行循环体
第二次: ipairs(arr, 1) → (2, "b") → 执行循环体
第三次: ipairs(arr, 2) → (3, "c") → 执行循环体
第四次: ipairs(arr, 3) → nil      → 结束
```

### 常见错误

```lua
-- ❌ 在迭代器中修改被遍历的 table
for i, v in ipairs(arr) do
    table.insert(arr, i, v * 2)  -- ❌ 可能导致无限循环或跳过元素
end

-- ✅ 正确的做法：收集到新 table
local doubled = {}
for i, v in ipairs(arr) do
    doubled[i] = v * 2
end
```

## 五语言对比

| 特性 | Lua | Python | Rust | Go | C++ |
|------|-----|--------|------|-----|-----|
| if | `if then end` | `if :` | `if {}` | `if {}` | `if {}` |
| switch/match | 无（表跳转） | `match`（3.10+） | `match` | `switch` | `switch` |
| while | `while do end` | `while :` | `while {}` | `for` | `while` |
| repeat | `repeat until` | 无 | 无 | 无 | `do-while` |
| for | 数值/泛型 | `for` | `for`/`while` | `for` | `for`/`while` |
| break | 有 | 有 | 有 | 有 | 有 |
| continue | 无（goto） | 有 | 有 | 无 | 有 |
| 异常处理 | pcall/xpcall | try/except | Result/panic | panic/recover | try/catch |

## 本章小结

| 特性 | Lua 行为 |
|------|---------|
| if | `if cond then ... end` |
| if-else | `if ... then ... else ... end` |
| if-elseif | `if ... then ... elseif ... then ... end` |
| 循环 while | `while cond do ... end` |
| 循环 repeat | `repeat ... until cond`（至少执行一次） |
| 循环 for | `for i=1,n do ... end` 或 `for k,v in iter do ... end` |
| break | 退出当前循环 |
| continue | Lua 无，需要用 if 或 goto（Lua 5.2+）模拟 |
| 错误处理 | `error`/`assert`/`pcall`/`xpcall` |
| 迭代器 | `ipairs`/`pairs`/`gmatch`/自定义 |
